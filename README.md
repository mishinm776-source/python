Техническая документация

«Личный финансовый учёт» 

1. Введение

1.1 Цель

Документ описывает архитектуру, установку, настройку и использование десктопного приложения для управления личными финансами.
Приложение решает проблему отсутствия прозрачного учёта доходов/расходов у частных лиц.

1.2 Область применения

· Ведение домашней бухгалтерии
· Анализ структуры расходов
· Контроль финансовых потоков в малом бизнесе (ИП, самозанятые)

1.3 Аудитория

· Конечные пользователи (физические лица)
· Технические специалисты (установка, настройка)
· Разработчики (модификация и расширение)

2. Обзор системы

2.1 Архитектура

Паттерн: MVC (Model-View-Controller)

```
┌─────────────────────────────────────────────┐
│                 main.py                     │
│           (точка входа, DI)                 │
└────────────┬────────────────────────────────┘
             │
    ┌────────┴────────┬──────────────┐
    ▼                 ▼              ▼
┌─────────┐     ┌──────────┐   ┌────────────┐
│ Models  │◄───►│Controller│   │   Views    │
│ (данные,│     │(логика)  │──►│ (GUI)      │
│  JSON)  │     │          │   │            │
└─────────┘     └──────────┘   └────────────┘
      ▲                               │
      └───────────┬───────────────────┘
                  ▼
           [ finance_data.json ]
```

Компоненты:

· Model – работа с данными (JSON-файл, бизнес-логика расчётов)
· View – tkinter-интерфейс, таблицы, графики
· Controller – обработка событий, синхронизация M и V

2.2 Используемые технологии

Компонент Технология Версия
GUI tkinter / ttk встроена в Python 3.x
Графики matplotlib ≥3.5.0
Хранение данных JSON (файловая система) -
Язык программирования Python ≥3.8

2.3 Зависимости

```bash
matplotlib>=3.5.0
python-dateutil>=2.8.0  # опционально
```

4. Руководство по настройке

4.1 Параметры конфигурации

Файл config.py (создаётся вручную при необходимости):

```python
# config.py
DATA_FILE = "finance_data.json"           # Имя файла данных
DEFAULT_CATEGORIES = ["Еда", "Транспорт", "ЖКХ", "Зарплата", 
                      "Развлечения", "Здоровье"]
BACKUP_ENABLED = True                     # Автоматическое резервирование
BACKUP_INTERVAL_DAYS = 7                  # Каждые 7 дней
```

4.2 Настройка окружения

Переменные окружения (опционально):

```bash
export FINANCE_DATA_PATH="/custom/path/data.json"
export FINANCE_LANG="ru_RU.UTF-8"
```

4.3 Интеграция внешних сервисов

Приложение не использует внешние API.
Для импорта банковских выписок требуется ручное преобразование в JSON.

Формат импорта:

```json
{
  "transactions": [...],
  "categories": [...]
}
```

7. Схема базы данных

7.1 Диаграмма «сущность-связь»

```
┌──────────────┐         ┌─────────────────┐
│  Transaction │         │   Category      │
├──────────────┤         ├─────────────────┤
│ id (PK)      │────────►│ name (PK)       │
│ date         │         └─────────────────┘
│ category (FK)│
│ amount       │         ┌─────────────────┐
│ type         │         │   User (план)   │
│ comment      │         ├─────────────────┤
└──────────────┘         │ id, settings    │
                         └─────────────────┘
```

7.2 Определения таблиц (JSON-схема)

Транзакция:

```json
{
  "id": "integer (auto-increment)",
  "date": "string (YYYY-MM-DD)",
  "category": "string",
  "amount": "float (>0)",
  "type": "enum('Доход', 'Расход')",
  "comment": "string (max 200 chars)"
}
```

Категория:

```json
{
  "categories": ["string"]
}
```

7.3 Ограничения и проверки

· amount > 0 (отрицательные значения запрещены)
· date в формате ISO 8601
· category должна существовать в списке категорий
· type только 'Доход' или 'Расход'

10. Поддержка и обслуживание

10.1 Руководство по устранению неполадок

Проблема: Приложение не запускается
Диагностика:

```bash
python main.py --debug
```

Решение: Установить недостающие пакеты или обновить Python.

Проблема: Потеря данных
Восстановление: Использовать последний экспортированный JSON → «Импорт».

10.2 Часто задаваемые вопросы (FAQs)

Q: Как перенести данные на другой компьютер?
A: Экспортировать в JSON → скопировать файл → импортировать.

Q: Можно ли изменить категории?
A: Да, через редактирование файла finance_data.json (раздел categories).

Q: Безопасны ли мои данные?
A: Данные хранятся локально, не передаются в интернет.

Q: Поддерживается ли валюта, отличная от рубля?
A: Формально – да, но символ валюты не отображается.
